# 使用说明

## 推荐输入

```text
使用智能推荐 Skill 分析这个需求，并生成可落到画布的产品 UX 方案。
需求：……
项目背景：……
已有知识/蓝图：……
约束：……
```

## 运行顺序

1. 读取用户输入、项目知识、上传文档和当前画布。
2. 判断需求意图和是否需要转交专用 Skill。
3. 按 `design-scheme-ux` 输出用户行为、功能层级、页面形态、路径返回和状态错误。
4. 生成前后端分工、接口契约和 UX 验收清单。
5. 如果用户补充确认，合并原画布并刷新完整方案。

## 检查清单

- 是否说明用户第一眼想看到什么。
- 是否说明哪些信息不展示或降级。
- 是否覆盖页面形态选择原因。
- 是否覆盖返回状态和草稿保留。
- 是否覆盖错误发生位置、影响和恢复动作。
- 是否输出前后端接口契约。
