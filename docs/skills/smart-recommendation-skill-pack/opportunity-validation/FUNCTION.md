# 功能说明

`opportunity-validation` 用于把项目资料、产品文档、蓝图、交互路径、Demo 和竞品流程转成可执行的产品 UX 迭代方案。它重点回答用户路径哪里卡住、竞品为什么这样设计、我们参考什么、不照搬什么、如何更好。

## 核心能力

- 建立项目基础信息：人群、目标、定位、当前阶段、主要任务。
- 模拟用户旅程：空输入、上传、解析、蓝图、Demo、失败恢复、导出、返回修改。
- 反向分析竞品流程：入口、输入、状态、确认、失败、产物沉淀和交付路径。
- 建立用户行为意图和设计方案决策矩阵。
- 结合 RICE、Kano、功能金字塔和竞品价值排序机会点。
- 做 UX、产品价值、工程可执行三轮评审。

## 输出产物

- 竞品流程矩阵。
- 用户行为意图矩阵。
- 当前需求证据地图。
- 设计方案决策矩阵。
- 机会点 Top。
- 差异化方案。
- 迭代计划和开发任务。
