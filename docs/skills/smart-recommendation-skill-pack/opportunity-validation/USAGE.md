# 使用说明

## 推荐输入

```text
使用 opportunity-validation Skill 对当前项目执行机会验证。
请结合项目资料、上传文档、蓝图、交互路径、Demo 和竞品流程，
输出用户行为意图、设计方案决策矩阵、机会点、竞品参考、差异化方案、优先级和开发任务。
```

## 运行顺序

1. 建立项目上下文。
2. 定义用户角色和 JTBD。
3. 分析用户行为意图。
4. 提取当前需求证据。
5. 建立设计方案决策矩阵。
6. 做竞品流程反向分析。
7. 模拟当前产品用户旅程。
8. 排序机会点。
9. 做三轮评审。
10. 输出迭代计划和开发任务。

## 检查清单

- 是否解释竞品为什么这样做。
- 是否说明参考什么、不照搬什么、我们怎么更好。
- 每个 P0/P1 机会点是否有当前产品路径证据。
- 每个方案是否说明承载形态、路径返回、错误恢复和验收方式。
