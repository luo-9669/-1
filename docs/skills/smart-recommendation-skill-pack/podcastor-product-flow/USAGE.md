# 使用说明

## 推荐输入

```text
使用 Podcastor 产品体验流，生成 AI 播客工作站的一期产品蓝图。
目标用户：……
核心痛点：……
输入方式：……
编辑器能力：……
交付目标：……
```

## 运行顺序

1. 做播客需求诊断。
2. 定义首页创作入口。
3. 拆核心编辑器流程。
4. 规划资产复用和分发。
5. 定义增长指标和验收标准。
6. 输出产品蓝图、页面清单、状态机和工程交付拆解。

## 检查清单

- 是否覆盖脚本、音频、视频三类输入。
- 是否说明生成、编辑、预览、导出和失败恢复。
- 是否区分 MVP 范围和后续范围。
- 是否能转给设计方案和工程开发继续执行。
