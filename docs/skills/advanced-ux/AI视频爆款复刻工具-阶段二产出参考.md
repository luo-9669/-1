## 1. 文档概述

### 1.1 文档目的

本文档基于阶段一 v3.0 的需求分析、方案设计和推荐决策，为一期交付的 15 个界面单元产出完整的页面级交互框架与说明。

**产出范围**：
- 页面总览表 + 信息架构实体表
- 主流程步骤表 + 页面流转图 + 页面间数据流
- 核心对象状态机（含完整迁移规则）
- 弹窗与抽屉定义表
- 逐页交互说明（8 个页面 × 框架表 + IR 规则 + 异常状态 + 线框图）
- 关键断点与优化节点
- 全局交互规范（7 类）
- 页面 3 方案（核心流程整体方案对比 + 加权评分）
- 方案验证与收敛（P-S Fit + 决策卡片 + 埋点 + 待确认）

### 1.2 编号规则

| 对象类型 | 编号格式 | 起始值 | 示例 | 说明 |
|----------|---------|--------|------|------|
| 页面/弹窗 | P + 两位数字 | PG01 | PG01, P1 | 弹窗共用 P 编号 |
| 状态机状态 | ST + 页面号-序号 | ST0-1 | ST2-3 | ST 后 = 页面编号，-后 = 序号 |
| 主流程步骤 | S + 数字 | S1 | S1, S7 | 按执行顺序递增 |
| 交互规则 | IR + 数字 | IR1 | IR1, IR10 | 按页面内顺序递增 |
| 异常状态 | E + 页面号-序号 | E1-1 | E5-3 | 按页面内顺序递增 |

### 1.3 编号联动规则

| 联动关系 | 说明 | 本项目示例 |
|----------|------|-----------|
| IR ↔ P 弹窗 | 交互规则触发弹窗时引用 P 编号 | PG01 IR1 → P1 解析进度弹窗 |
| IR ↔ ST 状态 | 交互规则触发状态变化时引用 ST 编号 | PG01 IR1 → ST0-1 迁移至 ST0-2 |
| 线框图 ↔ IR/P/ST | 线框图标注交互区域引用编号 | PG01 ①链接输入区 → IR1-IR3, P1 |
| S 步骤 ↔ P 页面 | 每个步骤指向对应页面 | S1 → PG01 → P1 → PG02 |

### 1.4 核心假设（继承自阶段一，含置信度）

1. **产品类型**：Web 工具（置信度：高，需求原文明确）
2. **核心场景**：发现爆款→AI 拆解结构→迁移内容→配置素材→生成同款（置信度：中，基于蝉镜/凡科/NemoVideo 竞品推断）
3. **目标用户**：电商运营（P0）/ 知识博主（P1）/ MCN 团队（P2）（置信度：中，基于蝉镜用户画像推断）
4. **MVP 范围**：单条复刻 5 步向导流程，不含批量生成（置信度：中，基于行业惯例推断）
5. **核心对象**：爆款视频 / 结构蓝图 / 分镜 / 定制脚本 / 素材槽位 / 数字人 / 生成任务 / 复刻视频 / 模板 / 用户项目（置信度：高，基于竞品功能拆解）

## 2. 页面总览

### 2.1 页面识别检查清单

- [x] 用户从打开到完成核心任务，经过的页面：PG01→PG02→PG03→PG04→PG05→PG06 = 6 个核心页面
- [x] 弹窗/浮层：P1 解析进度 / P2 素材上传 / P3 生成排队 / P4 导出设置 / P5 合规提醒 / P6 新手引导 = 6 个弹窗
- [x] 异常路径：无独立异常页（异常通过弹窗和页面内状态处理）
- [x] 辅助页面：PG07 灵感广场 / PG09 我的项目 = 2 个页面
- [x] 角色区分：一期不区分角色（所有用户相同流程）

## 10. 页面 3 方案

> 对核心流程的整体交互模式进行 3 套方案对比。PG05 生成预览页已融合方案 B 精修面板能力（见阶段一决策）。

### 10.1 方案 A（稳妥型）—— 线性步骤向导

**方案定位**：5 步线性向导，一步一页，每页只承载一个核心任务

**核心思路**：
- 5 步向导：导入→拆解→迁移→配置→生成，每步独立页面
- 步骤条始终可见，已完成步骤可点击回退
- AI 在每步自动填充默认值，用户只需确认或微调
- 在预览页提供基础精修能力（参数调节 / 局部修改）

**完整链路**：

| 步骤 | 节点 | 用户动作 | 系统反馈 | 跳转 |
|------|------|---------|---------|------|
| S1 | PG01 首页 | 粘贴链接/上传 | 进入解析，P1 弹窗 | → P1→PG02 |
| S2 | PG02 蓝图 | 查看拆解结果（三层渐进） | 展示摘要 + 分镜表 + 曲线 | → S3 |
| S3 | PG03 迁移 | 填写业务信息 | AI 生成定制脚本 | → S4 |
| S4 | PG04 配置 | 上传素材 | 匹配到槽位 + 缩略图 | → S5 |
| S5 | PG05 预览 | 生成→预览→微调 | AI 合成 + 播放 + 精修面板 | → S6 |
| S6 | PG06 结果 | 导出 | 下载视频 | 完成 |

**关键页面说明**：
- PG02：三层渐进展示（摘要→概览→详情），默认只展开 Layer1
- PG03：行业预设模板 + AI 辅助填充，减少填写门槛
- PG05：融合方案 B 精修面板，支持参数调节和局部修改

**优势**：
- 新手友好，认知负担最小
- 开发复杂度低，与凡科等已验证流程一致
- 每步可独立优化和 A/B 测试

**风险**：
- 步骤多，效率偏低
- 回退修改成本高（需跳回前面步骤）

**适用前提**：
- 用户以新手为主
- 一期追求快速交付

**置信度**：高

---

### 10.2 方案 B（效率型）—— 多面板工作台

**方案定位**：单页面多面板，所有步骤在一个工作台内完成

**核心思路**：
- 左侧为可折叠的流程面板（5 个面板上下排列），右侧为实时预览区
- 每个面板可独立展开/折叠，用户在同一页面内完成全流程
- 任意时刻都能看到视频预览效果（原视频/生成中/成品）
- 面板间自由跳转，无需线性前进
- AI 助手悬浮球在右下角，随时辅助

**完整链路**：

| 步骤 | 节点 | 用户动作 | 系统反馈 |
|------|------|---------|---------|
| S1 | 工作台-导入面板 | 粘贴链接 | 视频在下方展开 + 蓝图面板自动加载 |
| S2 | 工作台-蓝图面板 | 查看/编辑 | 分镜表实时可编辑 |
| S3 | 工作台-迁移面板 | 填写信息 | 脚本实时预览 |
| S4 | 工作台-配置面板 | 拖拽素材 | 预览区实时更新 |
| S5 | 工作台-生成面板 | 点击生成 | 生成结果在预览区播放 |
| S6 | 导出 | 点击导出 | 弹出导出设置 |

**关键页面说明**：
- 右侧预览区始终可见，从原视频到生成结果无缝切换
- 面板间数据自动传递，无需手动"下一步"
- 支持任意面板回退修改，数据不丢失

**优势**：
- 操作效率高，全局视野好
- 随时回看/修改，无需线性跳转

**风险**：
- 信息密度高，新手迷失
- 单页开发复杂度高
- 实时预览技术要求高

**适用前提**：
- 中高频用户为主
- 技术能力强

**置信度**：中

### 10.3 方案 C（引导型/增强型）—— AI Agent 对话式

**方案定位**：以 AI Agent 为核心驱动，对话式引导完成全流程

**核心思路**：
- 主界面为对话窗口，AI Agent 主动引导每一步
- 用户通过自然语言描述需求，AI 自动执行
- 结果在对话气泡中以富媒体形式展示（视频/表格/曲线图）
- 随时可用自然语言修改（"把第 3 段换成产品展示"）
- AI Agent 主动提供建议和优化方向

**完整链路**：

| 步骤 | 节点 | 用户动作 | 系统反馈 |
|------|------|---------|---------|
| S1 | 对话窗口 | 粘贴链接/"帮我复刻这个" | AI 自动解析 + 展示蓝图摘要 |
| S2 | 对话窗口 | "我想做面膜带货" | AI 生成定制脚本 + 展示 |
| S3 | 对话窗口 | 上传素材/"用这个产品图" | AI 匹配槽位 |
| S4 | 对话窗口 | "生成视频" | AI 生成 + 对话中展示预览 |
| S5 | 对话窗口 | "BGM 换欢快一点的" | AI 调整 + 重新生成 |

**关键页面说明**：
- 对话气泡内嵌富媒体卡片（视频播放器/分镜表/曲线图）
- AI 主动提示下一步操作
- 支持随时回溯修改

**优势**：
- 零学习成本
- 灵活度最高
- 差异化极强

**风险**：
- 依赖 AI 理解能力
- 对话效率低于表单

**适用前提**：
- AI 能力成熟
- 作为二期高级功能

**置信度**：低

### 10.6 方案选择建议

**推荐方案**：方案 A（稳妥型）+ 方案 B 部分能力融合

**推荐理由**：
- 蝉镜核心用户（电商运营/知识博主）以新手为主，向导式降低门槛
- 在预览页融合方案 B 的实时精修能力，满足进阶需求
- 开发复杂度可控，一期可交付

**为什么不优先选其他方案**：
- 方案 B 未被优先：信息密度高，新手容易迷失，一期开发成本偏高
- 方案 C 未被优先：效率取决于 AI 理解能力，难以精细控制，对话式交互效率偏低

**融合建议**：
- 在 S6 预览调整阶段，提供方案 B 的"参数调节 + 局部修改"面板，而非跳回前面步骤
- 二期将方案 C 的 AI Agent 能力嵌入 PG03 脚本定制和 PG04 素材配置，作为智能助手角色

**需要确认的问题**：
